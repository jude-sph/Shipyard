You are generating Arcadia End Product Breakdown Structure (EPBS) elements for the Capella modeling tool.

PURPOSE OF THIS STAGE:
The EPBS (End Product Breakdown Structure) describes the structure of configuration items constituting the final solution. The organisation of the System-of-Interest (end product components) in a tree frame is the End Product Breakdown Structure. The EPBS is the part of the PBS related to the System under consideration in the development process. It links deliverable configuration items back to physical components defined in the Physical Architecture.

ARCADIA ELEMENT DEFINITIONS:

1. Configuration Items (configuration_items)
   A Configuration Item is any hardware, software, or combination of both that satisfies an end use function and is designated for separate configuration management. Configuration items are typically referred to by an alphanumeric identifier which also serves as the unchanging base for the assignment of serial numbers to uniquely identify individual units of the CI.
   Fields: id, name, type ("HW", "SW", "FW", "DATA", or "SERVICE"), description (string), physical_component_refs (list of physical component IDs from PA that this CI encompasses)

2. PBS Structure Nodes (pbs_structure)
   PBS nodes form a hierarchical tree that organises configuration items into a product breakdown structure. Each node can have a parent and children, forming the decomposition tree.
   Fields: id, name, parent_id (parent node ID, or null for root nodes), children_ids (list of child node IDs), ci_ref (configuration item ID that this node represents, empty string if purely structural)
