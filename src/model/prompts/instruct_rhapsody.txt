The instructions must be precise enough for a systems engineer to follow without prior Rhapsody experience. Reference actual Rhapsody menu paths, dialog names, toolbar actions, and the SysML profile elements.

RHAPSODY TOOL REFERENCE

Use these actual IBM Rhapsody 10.0 menu paths and actions in your steps:

Project creation:
- File > New > Project — opens the New Project dialog
- Select "SysML" in the Profile dropdown, set project name, click OK
- The project opens in the Model Browser on the left panel

Requirements Diagram:
- In Model Browser, right-click the package > Add New > Requirement — creates a SysML Requirement
- Right-click diagram area > Add New Diagram > SysML Requirements Diagram
- Drag requirements from Model Browser onto the diagram canvas
- Right-click a requirement > Features > Tags tab — set Id, Text, and custom tags

Block Definition Diagram (BDD):
- Right-click a package > Add New Diagram > SysML Block Definition Diagram
- Right-click diagram canvas > Add New > Block — creates a SysML Block
- Right-click a block > Features > Properties tab > Add > Property — adds typed properties
- Drag from toolbar: Association, Composition, Aggregation to connect blocks
- Right-click block > Add New > Port — creates flow ports

Internal Block Diagram (IBD):
- Right-click a block in Model Browser > Add New Diagram > SysML Internal Block Diagram
- Drag block properties (part associations) onto the canvas
- Connect ports using the Connector tool in the diagram toolbar

Activity Diagram:
- Right-click a block or package > Add New Diagram > Activity Diagram
- Drag Action nodes from toolbar onto canvas
- Connect with Control Flow arrows
- Right-click > Add New > ObjectNode for data flows

Sequence Diagram:
- Right-click a package > Add New Diagram > Sequence Diagram
- Drag lifelines from block instances in Model Browser
- Right-click between lifelines > Add Message — creates synchronous or asynchronous messages

State Machine Diagram:
- Right-click a block > Add New Diagram > Statechart
- Drag State from toolbar, set name in Properties dialog
- Right-click between states > Add New > Transition — connect states with triggers/guards

Traceability links:
- Right-click a block > Add New > SatisfyRelation — links a block to a requirement
- Right-click > Add New > DeriveReqt — links a derived requirement to a source
- Right-click > Add New > AllocateActivityPartition — allocates activity to block
- Use Tools > Requirements Traceability Matrix — view all satisfy/refine/trace links
